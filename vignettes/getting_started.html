Getting Started

Author

Akash B V and David LeBauer

Published

May 28, 2026

caladaptaer reads downscaled CMIP6 climate projections off the Cal-Adapt Analytics Engine directly into R. Data lives as cloud optimized Zarr on AWS S3, and the package reads it through GDAL: no Python, no API key, no authentication. Every call hands back a native R object, a stars grid or a tidy data.frame, so results drop right into the R geospatial and modeling stack.

This page goes from install to a real time series in a few calls, then shows the design choice that lets the same code scale to thousands of sites. Every output below comes from actually running the code on this page. For the full function tour see Accessing Cal-Adapt Data in R.

Install

remotes::install_github("lebauerapproach/caladaptaer")

You need R >= 4.1 and GDAL >= 3.4 for the Zarr driver. Confirm your GDAL is recent enough:

sf::sf_extSoftVersion()["GDAL"]
#>     GDAL 
#> "3.11.5"
library(caladaptaer)

Discover what’s available

Ask what the WRF dynamical downscaling covers:

cae_models(activity = "WRF")
#> [1] "CESM2"         "CNRM-ESM2-1"   "EC-Earth3"     "EC-Earth3-Veg"
#> [5] "ERA5"          "FGOALS-g3"     "MIROC6"        "MPI-ESM1-2-HR"
#> [9] "TaiESM1"

Eight GCMs plus the ERA5 reanalysis driven run. CESM2 carries the widest set of scenarios:

cae_scenarios(activity = "WRF", model = "CESM2")
#> [1] "historical" "ssp245"     "ssp370"     "ssp585"

Pull a time series at a point

cae_fetch_point() reads the grid and extracts the nearest cell to a lon/lat. Transform from WGS84 to the WRF Lambert Conformal grid happens for you. Here’s a summer day in 2050 at Fresno under SSP3-7.0:

ts <- cae_fetch_point("t2", model = "CESM2", scenario = "ssp370",
                      lon = -119.77, lat = 36.75,
                      start_time = "2050-07-01T00:00:00",
                      end_time   = "2050-07-01T23:00:00")
head(ts)
#>                  time    value
#> 1 2050-07-01 00:00:00 304.5662
#> 2 2050-07-01 01:00:00 302.7805
#> 3 2050-07-01 02:00:00 301.1025
#> 4 2050-07-01 03:00:00 298.4392
#> 5 2050-07-01 04:00:00 295.5602
#> 6 2050-07-01 05:00:00 293.1013

Result is a plain data.frame with time and value in Kelvin, so base R plotting just works. Times are UTC, so the midday dip is the local overnight low (UTC noon is roughly 5am Pacific).

ts$temp_c <- ts$value - 273.15
plot(ts$time, ts$temp_c, type = "l", lwd = 2, col = "#D62728",
     xlab = "time", ylab = "temperature (C)")

Extract many locations in one read

This is the design choice worth knowing up front. cae_fetch_points() reads the grid from S3 once and extracts every location locally in a single vectorized pass. Three sites or ten thousand, it’s one S3 read per variable, not one per site.

sites <- data.frame(
  site_id = c("davis", "fresno", "sacramento"),
  lon = c(-121.74, -119.77, -121.49),
  lat = c(38.54, 36.75, 38.58)
)
mp <- cae_fetch_points("t2", model = "CESM2", scenario = "ssp370",
                       points = sites,
                       start_time = "2050-07-01T00:00:00",
                       end_time   = "2050-07-01T05:00:00")
#> Reading grid (1 S3 fetch for all 3 sites)
#> Extracting 3 sites
head(mp, 8)
#>   site_id                time    value
#> 1   davis 2050-07-01 00:00:00 304.7827
#> 2   davis 2050-07-01 01:00:00 303.4674
#> 3   davis 2050-07-01 02:00:00 301.8932
#> 4   davis 2050-07-01 03:00:00 299.8713
#> 5   davis 2050-07-01 04:00:00 297.6558
#> 6   davis 2050-07-01 05:00:00 295.5995
